EXECUTIVE SUMMARY

01. 제품 개요

OfficeBridge는 VPN을 대체하는 클라이언트리스 제로트러스트 게이트웨이입니다. 직원은 브라우저만으로 인가된 사내 시스템(ERP·그룹웨어·GitLab 등)에 접속하고, 사내에 설치한 커넥터가 아웃바운드 터널로 연결해 방화벽 인바운드 개방이 전혀 필요 없습니다. 부서·개인별 접근 정책, 로그인 이후에도 매 요청을 재검증하는 행위 기반 지속 검증(Continuous Verification), 실시간 감사 로그와 계정 즉시 차단을 관리자 콘솔에서 통합 관리합니다.

한 줄 정의

“VPN 없이, 브라우저만으로, 인가된 사내 시스템에만 접근하게 하는 제로트러스트 게이트웨이”

왜 지금인가

  • 재택·외근·협력사 접근이 일상화되며 경계(perimeter) 기반 보안 모델이 붕괴 — 글로벌은 VPN에서 ZTNA로 이동 중
  • 최근 대형 침해사고 상당수가 인터넷에 노출된 VPN 장비의 취약점에서 시작 (Ivanti·Fortinet 등)
  • 국내 중소기업은 방화벽·VPN 장비와 보안 인력을 갖추기 어려워, 포트포워딩 등 무방비 상태로 사내 시스템을 노출 — 랜섬웨어의 주요 침투 경로
  • 글로벌 ZTNA(Cloudflare·Zscaler)는 해외 SaaS라 데이터 주권·언어·가격 장벽으로 국내 중소기업이 도입하기 어려움

PROBLEM

02. 문제 정의 — VPN의 한계

기존 원격 접속의 표준인 VPN은 “한 번 접속하면 사내망 전체를 신뢰”하는 구조로, 다음과 같은 근본적 한계를 가집니다.

문제내용
과도한 접근 권한VPN 접속 = 사내망 전체 진입. 계정 하나만 탈취되어도 내부 이동(lateral movement)으로 전산망 전체가 위험에 노출
공격 표면 노출VPN 장비가 인터넷에 직접 노출. 장비 취약점이 곧 침해 사고의 진입점 (CVE 단골 대상)
정적 신뢰로그인 한 번 성공하면 세션 내내 신뢰 — 세션 탈취·내부자 이상행위에 무방비
운영 부담전 직원 PC에 클라이언트 설치·업데이트·장애 대응이 필요
느린 권한 회수퇴사자·사고 계정의 접근 차단이 즉각적이지 않아 감사 지적사항으로 이어짐

우리의 진짜 경쟁 상대는 “무방비”입니다

방화벽·VPN 장비를 살 예산과 보안 인력이 없는 중소기업은 대부분 공유기 포트포워딩으로 NAS·원격데스크톱(RDP)·ERP 서버를 인터넷에 그대로 노출하고 비밀번호 하나로 방어하고 있습니다. 국내 랜섬웨어 감염의 주요 경로가 바로 이 “인터넷에 노출된 원격 접속”입니다. OfficeBridge의 경쟁 상대는 값비싼 글로벌 ZTNA가 아니라, 지킬 수단이 없는 현재의 무방비 상태입니다.

타깃 고객

  • 1차 타깃 — 사내에 서버(ERP·NAS·개발서버·생산관리)를 두어야 하지만, 보안 장비와 전담 인력이 없는 국내 중소기업 (제조·개발 외주사·병원·회계사무소 등)
  • 관리자 페르소나 — 전산을 겸직하는 총무 담당자. 전문 보안 지식 없이도 “눈에 보이는 통제”와 “쉬운 도입”을 원함
  • 사용자 페르소나 — 외근·재택 중 노트북으로 사내 시스템에 접속해야 하는 직원. VPN 앱 설치·끊김에 스트레스

SOLUTION

03. 솔루션 & 작동 원리

OfficeBridge는 릴레이 서버(벤더 운영)와 커넥터(고객사 설치)로 구성됩니다. 핵심은 커넥터가 사내에서 밖으로(아웃바운드) 터널을 열어두는 구조로, 고객사 방화벽에 어떤 인바운드 포트도 열 필요가 없다는 점입니다.

외부 직원 브라우저→ 443 → 릴레이 (벤더 IDC)← 터널 ← 커넥터 (고객사)→ 사내 시스템

요청 처리 4단계 검문

① 인증 (누구?)→ ② 정책 (되는 앱?)→ ③ 행위 검증 (지금 안전?)→ ④ 터널 전달

인증된 세션이라도 매 요청마다 ①~③을 다시 통과해야만 사내 시스템에 도달합니다. 차단된 요청은 사내망에 패킷조차 도달하지 않습니다.

FEATURES

04. 핵심 기능 (구현 완료)

클라이언트리스 접속

직원은 브라우저로 대표 포털에 접속해 본인에게 허가된 시스템만 확인·이용. 권한 없는 시스템은 잠금 표시.

아웃바운드 터널

커넥터가 릴레이로 상시 아웃바운드 연결을 유지. 방화벽 인바운드 개방 불필요, 자동 재연결.

조직도 기반 정책

부서 단위로 접근 권한을 부여하면 소속 직원 전원에 적용. 개인별 추가 권한도 부여 가능. 즉시 반영.

행위 기반 지속 검증

업무시간 외 차단·요청 폭주 차단·세션 IP 고정·로그인 실패 잠금·세션 수명 제한 등 룰셋을 매 요청 재검증.

실시간 감사 로그

누가·언제·어느 시스템에·어떤 판정으로 접근했는지 전량 기록. 허용/차단 필터, 관리자 조치 이력 별도 관리.

계정·세션 즉시 통제

활성 세션 강제 종료, 계정 차단 시 전 세션 폐기 + 재로그인 거부. 퇴사자 권한 회수 즉시.

관리자 콘솔

대시보드 · 사내시스템 등록 · 정책 접근관리 · 행위 기반 제어 · 활성세션/계정통제 · 시스템 접속로그 · 관리자 로그 — 7개 메뉴로 전 기능을 웹에서 통합 관리. 사내 시스템 등록은 라벨·내부주소 입력만으로 1분 내 온보딩.

도입 절차 ① 가입·커넥터 발급 → ② 사내 서버에 커넥터 실행 → ③ 사내 시스템 등록 → ④ 부서별 정책 설정 → ⑤ 직원에게 포털 주소만 공지. 방화벽 변경 0건, 직원 PC 설치 0건.

07. 향후 개발 로드맵

1. 지란지교소프트 서비스와의 연결

1-1) 에이전트 기반 배포 + 조직도 공유

고객사는 이미 지란지교 제품 사용을 위한 SW를 PC마다 배포·운영하고 있으며, 조직 관리자가 구성한 조직도를 사용합니다. 현재의 커넥터 기반을 에이전트 기반으로 확장해 우리 제품과 함께 배포하면, 동일한 조직도를 그대로 재사용할 수 있습니다. 별도 사용자·조직 관리 없이 즉시 정책 연동이 가능합니다.

1-2) PC 보안 점수 기반 접근 제어 (OfficeKeeper 연동)

지란지교 제품은 PC 취약점 점검 기능을 제공하여, 관리자가 지정한 정책에 따라 PC 보안 점수를 산출하고 단말의 안전 여부를 판단합니다. 이를 OfficeBridge에 연동해 안전한 PC(기준 점수 이상)에서만 접속을 허용하도록 구성할 수 있습니다. 백신 미설치·미조치 취약점이 있는 기기의 사내 접근을 원천 차단합니다. (관리자 콘솔 UI 반영 완료, 판정 로직 연동 예정)

2. 사용자 계정 보안 강화 — MFA

비밀번호 인증에 더해 다중 인증(MFA)을 적용하여 계정 탈취 위험을 차단합니다. 로그인 시 추가 인증 수단(OTP·인증 앱 등)으로 본인 확인을 강화하여, 자격증명 유출만으로는 접근할 수 없도록 안전성을 보장합니다.

3. GeoIP 기반 해외 접속 차단

GeoIP 식별을 통해 접속 IP의 국가를 판별하고, 국내에서 근무하는 조직 특성상 불필요한 해외 IP의 로그인 시도를 차단합니다. 해외 기반 자동화 공격·자격증명 대입 시도를 원천 차단하여 공격 표면을 크게 줄입니다.

기타 확장

  • 엔드포인트 에이전트를 통한 비웹 프로토콜(RDP·SSH·NAS) 지원
  • 기존 SSO(OIDC) 연동 — Google Workspace·MS 365 등
  • 커스텀 도메인·전용 릴레이(온프레미스) 옵션
