# PRD — OfficeBridge

> **OfficeBridge** — 클라이언트리스 ZTNA 게이트웨이
> 버전: v0.2 (2026-07-09) · 해커톤 24시간 MVP 기준
> 상태: 초안 — 팀 리뷰 중

---

## 1. 한 줄 정의

**VPN 없이, 브라우저만으로, 인가된 사내 시스템에만 접근하게 하는 제로트러스트 게이트웨이.**
AI 정책 엔진이 모든 접속의 위험도를 실시간 평가한다.

## 2. 문제 정의

### 기존 방식 (VPN)의 문제

| 문제 | 설명 |
|---|---|
| 과도한 접근 권한 | VPN 접속 = 사내망 **전체** 진입. 계정 하나 탈취되면 내부 이동(lateral movement)으로 전산망 전체가 위험 |
| 공격 표면 노출 | VPN 장비가 인터넷에 직접 노출. 최근 수년간 대형 침해사고의 단골 진입점 (Ivanti, Fortinet 등 VPN 취약점) |
| 정적 신뢰 | 로그인 한 번 성공하면 세션 내내 신뢰. 세션 탈취·내부자 이상행위에 무방비 |
| 운영 부담 | 전 직원 PC에 클라이언트 설치·업데이트·장애 대응 |
| 느린 권한 회수 | 퇴사자·사고 계정의 접근 차단이 즉각적이지 않음 |

### 왜 지금인가

- 재택·외근·협력사 접근이 일상화 — 경계(perimeter) 기반 보안 모델 붕괴
- 글로벌은 ZTNA(Cloudflare Access, Zscaler, Tailscale)로 이동 중이나, **국내는 데이터 주권·컴플라이언스 이슈로 해외 SaaS 도입 장벽** → 국내 IDC 기반 서비스의 기회

## 3. 목표 / 비목표

### 해커톤 목표 (24h)

1. 외부 사용자가 **브라우저만으로** SSO 인증 후 사내 웹 시스템에 접속하는 E2E 데모
2. 정책 기반 접근 제어 (허용 앱 / 차단 앱)
3. **AI 위험도 평가** — 접속 컨텍스트를 LLM이 평가, 고위험 시 차단
4. 관리자 대시보드 — 실시간 접속 로그 + **원클릭 즉시 차단**
5. 발표자료 완성

### 비목표 (명시적으로 안 만드는 것)

- PC 에이전트 (비웹 프로토콜 TCP/UDP 터널링) → 로드맵 2단계
- 기기 신뢰도 검증 (백신/OS 패치 상태) → 로드맵 2단계
- 고가용성, 성능 최적화, 커넥터 이중화
- 자체 계정 체계 (SSO는 Google OAuth로 대체)
- 실제 IDC 배포 (데모는 로컬 또는 클라우드 VM 1대)

## 4. 타깃 고객 & 페르소나

- **1차 타깃:** 임직원 50~1,000명 규모, 웹 기반 전산시스템(그룹웨어·ERP·위키) 중심의 국내 중견기업. 보안팀이 있으나 소수 정예
- **관리자 페르소나 "보안팀 박 과장":** VPN 장비 CVE 대응에 지침. 퇴사자 계정 회수 누락이 감사 지적사항. 도입이 쉽고 눈에 보이는 통제를 원함
- **사용자 페르소나 "영업팀 김 대리":** 외근 중 노트북·태블릿으로 그룹웨어 접속. VPN 앱이 자꾸 끊겨서 스트레스

## 5. 제품 구성 (아키텍처)

```
[외부 사용자 브라우저] ──HTTPS──▶ [릴레이 서버 (자사 IDC)]
                                   ├ 게이트웨이 프록시
                                   ├ 인증 모듈 (SSO)
                                   ├ 정책 엔진 (정적 룰 + AI 위험도)
                                   ├ 터널 매니저
                                   ├ 감사 로그 (SQLite)
                                   └ 관리자 대시보드
                                        ▲
                                        │ 아웃바운드 WSS 터널 (인바운드 개방 0)
                               [커넥터 (고객사 사내망, 1대)]
                                        │
                               [사내 웹 시스템들 (데모: 목업)]
```

- 사용자 PC: 설치물 없음 (클라이언트리스)
- 커넥터: 고객사당 1대, 사전 발급 토큰으로 릴레이에 등록 (멀티테넌트)

## 6. 기능 요구사항

### P0 — 데모에 반드시 필요

| ID | 기능 | 상세 |
|---|---|---|
| F-1 | 터널링 | 커넥터가 릴레이로 아웃바운드 WSS 연결 수립·유지·자동 재연결. 릴레이는 HTTP 요청/응답을 터널 위로 중계 |
| F-2 | SSO 인증 | 미인증 요청 → Google OAuth 로그인 → 세션 쿠키 발급 |
| F-3 | 정적 정책 | 사용자(이메일)↔앱 매핑 테이블. 미허용 앱 접근 시 403 차단 화면 |
| F-4 | AI 위험도 평가 | 접속 컨텍스트(시간대, 접속 이력 대비 이상 여부, 요청 빈도)를 Claude API로 평가 → 0~100 점수. 임계값 초과 시 차단 + 사유 표시 |
| F-5 | 감사 로그 | 모든 요청 기록: 사용자, 시각, 대상 앱, 판정(허용/차단), 위험도 점수, 사유 |
| F-6 | 관리자 대시보드 | 실시간 로그 스트림(SSE), 활성 세션 목록, 사용자 **즉시 차단** 버튼(세션 폐기) |
| F-7 | 사내 시스템 목업 | 그룹웨어·위키·인사DB 3종. 그럴듯한 UI (데모 무대장치) |

### P1 — 시간 남으면

| ID | 기능 | 상세 |
|---|---|---|
| F-8 | 고위험 시 추가 인증 | 차단 대신 step-up 인증(이메일 OTP 등) 요구 |
| F-9 | AI 이상행위 요약 | 대시보드에서 "오늘의 이상 접속 요약"을 LLM이 자연어로 리포트 |
| F-10 | 커넥터 상태 모니터링 | 대시보드에 커넥터 연결 상태·지연시간 표시 |

## 7. 데모 시나리오 (발표 2분)

1. **[정상 접속]** 외부망(테더링) 노트북에서 그룹웨어 URL 접속 → Google 로그인 → 그룹웨어 화면. *"VPN 없이요. 설치한 것도 없습니다."*
2. **[정책 차단]** 같은 사용자가 인사DB URL 접속 → 403 차단 화면. *"권한이 없는 시스템은 존재조차 보이지 않습니다."*
3. **[AI 차단]** 시간 조작으로 "새벽 3시 + 대량 요청" 상황 재현 → AI 위험도 87점 → 차단 + 대시보드 경고. *"룰에 없는 이상행위도 AI가 잡습니다."*
4. **[즉시 차단]** 관리자가 대시보드에서 차단 버튼 클릭 → 사용자 화면 즉시 세션 종료. *"퇴사자 권한 회수, 1초입니다."*

## 9. 차별점 (심사 Q&A 방어 논리)

1. **AI 동적 정책 엔진** — 기존 ZTNA는 정적 룰. 우리는 접속 컨텍스트를 LLM이 평가해 위험도 기반으로 동적 판단
2. **국내 IDC 기반** — 트래픽이 해외 클라우드를 경유하지 않음. 데이터 주권·컴플라이언스 민감 고객(금융·공공·제조) 공략
3. **도입 마찰 최소** — 고객사 방화벽 변경 0 (아웃바운드 443만), 직원 PC 설치물 0

## 10. 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 릴레이 서버 | Node.js (Express + ws) | 터널 매니저 포함 |
| 커넥터 | Node.js 단일 스크립트 | http-proxy로 내부 전달 |
| 대시보드 | React (Vite) + SSE | |
| 목업 앱 | 정적 HTML 또는 Express 3개 | |
| AI | Claude API | 위험도 평가 프롬프트 |
| DB | SQLite | 세션·정책·로그 |
| 인증 | Google OAuth 2.0 | |

## 11. 리스크

| 리스크 | 대응 |
|---|---|
| 터널 매니저(F-1)가 기술적 최대 난관 | **시작 직후 최우선 착수.** 4시간 내 관통 못 하면 플랜B: 릴레이→커넥터 직접 HTTP 프록시로 단순화(데모 무대에선 구분 안 됨) |
| Google OAuth 콜백 설정 이슈 | 초반에 등록·테스트. 안 되면 데모용 매직링크 로그인으로 대체 |
| AI 평가 지연으로 접속 느려짐 | 위험도 평가를 비동기·캐시 처리. 데모 시나리오 구간만 동기 평가 |
| 발표자료 뒷전으로 밀림 | 마감 3시간 전 코드 프리즈, 발표·리허설 전환 |
